# Docker Lab: Веб-приложение с Flask + Nginx + PostgreSQL + Redis

## Описание проекта

Учебный проект по контейнеризации трехуровневого веб-приложения. Проект демонстрирует использование Docker и Docker Compose для развертывания полноценного приложения с бэкендом, фронтендом, базой данных и системой кеширования.

### Функциональность
- Веб-интерфейс для управления списком задач (To-Do list)
- REST API с операциями CRUD (создание, чтение, обновление, удаление)
- Постоянное хранение данных в PostgreSQL
- Кеширование списка задач в Redis для оптимизации производительности
- Проксирование API запросов через Nginx

## Архитектура и схема взаимодействия

Проект состоит из четырех сервисов, которые взаимодействуют между собой внутри общей сети Docker:

- **frontend** (Nginx) — принимает запросы от браузера на порту 80, отдает статические файлы и перенаправляет API-запросы на backend
- **backend** (Flask) — обрабатывает API-запросы, получает данные из PostgreSQL и кеширует список задач в Redis
- **postgres** (PostgreSQL) — хранит все задачи в таблице tasks
- **redis** (Redis) — хранит закешированный список задач в течение 30 секунд

Все сервисы общаются друг с другом по именам: `backend` обращается к `postgres:5432` и `redis:6379`, а `frontend` проксирует запросы на `backend:5000`. Только frontend доступен снаружи (порт 80), остальные сервисы изолированы внутри сети Docker.

## Инструкция по запуску

```bash
# 1. Клонировать репозиторий
git clone <url-репозитория>
cd docker-lab

# 2. Создать .env файл
cp .env.example .env
# Отредактируйте .env, установите свой пароль

# 3. Запустить все сервисы
docker compose up -d --build

# 4. Открыть в браузере
open http://localhost
```

## Описание переменных окружения

| Переменная | Описание | Значение по умолчанию |
|------------|----------|----------------------|
| `POSTGRES_DB` | Название базы данных PostgreSQL | taskdb |
| `POSTGRES_USER` | Имя пользователя PostgreSQL | appuser |
| `POSTGRES_PASSWORD` | Пароль пользователя PostgreSQL | changeme |

## Список полезных команд

```bash
# Запуск и остановка
docker compose up -d          # Запустить все сервисы
docker compose down           # Остановить сервисы
docker compose down -v        # Остановить и удалить тома
docker compose ps             # Статус сервисов

# Логи
docker compose logs -f        # Логи всех сервисов
docker compose logs backend   # Логи конкретного сервиса

# Вход в контейнеры
docker compose exec backend sh
docker compose exec postgres psql -U appuser -d taskdb
docker compose exec redis redis-cli
```

## Ответ на вопрос

Команда `docker compose down -v` останавливает и удаляет контейнеры, а также удаляет все тома с данными. В нашем проекте это означает, что удалится том `pgdata` с задачами из PostgreSQL. При следующем запуске база данных будет пустой. Без флага `-v` данные сохраняются.
